# Project Overview

## 1. 목적
Android logcat 로그를 기반으로, 사람이 직접 숫자를 해석하지 않고도  
변화 패턴을 직관적으로 분석할 수 있도록 시각화 툴을 제공한다.

## 2. 문제 정의
- logcat 로그는 텍스트 기반이며 가독성이 낮다.
- 특정 parameter의 시간 변화 추적이 어렵다.
- 로그 포맷이 고정되어 있지 않아 재사용 가능한 분석 도구가 부족하다.

## 3. 해결 방식
- 사용자가 logcat 로그를 복사하여 툴에 붙여넣는다.
- 선택한 분석 프로파일에 따라 로그를 파싱한다.
- timestamp 기반으로 데이터를 정렬한다.
- 프로파일에 정의된 방식으로 그래프를 생성한다.

## 4. 대상 로그 특징
- logcat 한 줄 단위 텍스트
- timestamp 포함
- keyword 기반 로그 구분
- 비정형 문자열 (JSON 아님)
- 로그 포맷 변경 가능

## 5. 핵심 개념
- Parsing Rule = Analysis Profile
- 하나의 프로파일이 다음을 정의:
  - 필터링 규칙
  - 파싱 규칙
  - 시각화 방식

## 6. 사용자
- Android middleware / application 개발자
- 로그 기반 디버깅 및 튜닝 엔지니어

## 7. 기대 효과
- 로그 분석 시간 단축
- 시각적 패턴 기반 문제 분석
- 반복 분석 작업 자동화
